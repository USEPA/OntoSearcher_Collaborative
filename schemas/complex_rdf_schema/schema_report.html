Schema Diagram Explanation

The RDF schema represents a complex graph with the following key elements:

  • Primary Entities: Main objects in the data model with high connectivity
  • Secondary Entities: Supporting objects with fewer connections
  • Link Entities: Objects that primarily serve to connect other entities
  • Nodebags: Blank nodes that group related properties (similar to embedded documents or complex attributes)
  • Property Tables: Sets of properties that commonly appear together, suggesting a database table pattern

SPARQL Query Patterns

Querying Nodebags:

SELECT ?entity ?prop1Value ?prop2Value
WHERE {
    ?entity ?reference ?nodebag .
    ?nodebag <property1> ?prop1Value ;
             <property2> ?prop2Value .
}

Finding Primary Entities with their Related Secondary Entities:

SELECT ?primary ?secondary ?relationship
WHERE {
    ?primary a <PrimaryClass> .
    ?secondary a <SecondaryClass> .
    ?primary ?relationship ?secondary .
}
